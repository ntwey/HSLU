# Lab: How to build a Security Operations Center (SOC)

## 1. Was ist ein SOC?
Eine operative Einheit zur Gewährleistung von Echtzeit-Situationsbewusstsein über den Sicherheitsstatus eines Netzwerks und zur Reaktion auf bestätigte Cyber-Vorfälle.
- **Business Driver:** Vorfälle, IP-Schutz, regulatorische Anforderungen (z.B. Banken) oder Drittanbieter-Assessments.

## 2. Komponenten eines SOC
- **People:** Analysten (L1-L3), Cyber Engineering, Use Case Developer, SOC Manager.
- **Processes:** Incident Handling, Vulnerability Management, Threat Intelligence, Compliance.
- **Technologies:** Logs, SIEM, Ticketing-Systeme, SOAR, EDR.

## 3. Implementierungsphasen
Die Umsetzung dauert typischerweise 6-12 Monate:
1. Planning (Mission & Strategie).
2. Organisational Structuring (Rollen & Team).
3. Process Development (IR & Engineering).
4. Technology Deployment (SIEM & Log-Infrastruktur).

## 4. SOC Lab Setup
Anforderungen an ein Lab: Leicht bereitstellbar, kosteneffektiv, virtuelle Maschinen inklusive SIEM.
- **Infrastructure as Code (IaC):** Verwendung von Tools wie Azure Bicep oder Terraform zur automatisierten Bereitstellung.
- **Cloud vs. On-Premise:** Cloud bietet einfache Skalierbarkeit, verursacht aber laufende Kosten.

## 5. Kern-Technologien im Lab
- **Sysmon (System Monitor):** Mächtiges Tool von Microsoft zur Aufzeichnung von Prozessstarts, Netzwerkverbindungen und Dateiänderungen auf Endpunkten.
- **Azure Sentinel:** Cloud-natives SIEM von Microsoft. Nutzt die Kusto Query Language (KQL) für Analysen.
- **Parsing:** Sysmon-Daten werden standardmässig als XML geliefert und müssen für Analysten via KQL-Funktionen lesbar gemacht ("geparst") werden.

## Wichtigste Key Takeaways
- Ein SOC besteht aus dem Dreiklang: People, Process, Technology.
- Vertrauen innerhalb des SOC-Teams basiert auf Kompetenz, Authentizität und Transparenz.
- Infrastructure as Code (IaC) ist der Standard, um konsistente Analyseumgebungen zu schaffen.
- Effektives Monitoring beginnt auf dem Endpunkt (Endpoint Visibility via Sysmon).
- Ohne ordentliches Parsing sind Logdaten für Analysten kaum nutzbar.
